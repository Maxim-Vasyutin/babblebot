# Автономная сборка: что можно, что нельзя

Проект собирается автономно, пока владельца нет за компьютером. Поэтому Claude Code
не должен упираться в шаги, требующие реальных аккаунтов или интерактивного ввода.

## Claude Code ДЕЛАЕТ сам
- Пишет весь код (route handlers, handlers, lib, схемы, тесты)
- Кладёт SQL-миграции из SPEC.md Блок 2 в supabase/migrations/
- Гоняет `npm run build`, `npm run lint`, юнит-тесты
- Генерирует README.md с чеклистом ручного запуска (Приложение А SPEC.md),
  подставляя плейсхолдеры туда, где нужны реальные токены
- Генерирует .env.example со всеми переменными из Блока 5

## Claude Code НЕ делает (выносит в README чеклистом)
- Создание Supabase-проекта, выполнение SQL в дашборде
- Vercel deploy, задание env в Vercel
- Установку webhook (`curl .../setWebhook` с реальным токеном) — только записать
  команду в README с плейсхолдерами <TOKEN>, <SECRET>, <APP_URL>. НЕ выполнять
- `/init` в рабочей группе
- Любые сетевые вызовы с реальными секретами

## Если упёрся
Не останавливайся и не жди ввода. Реализуй код так, чтобы он собирался с заглушками/
типами, а реальный шаг опиши в README. Прогресс важнее — владелец вернётся к готовому
коду и чеклисту, а не к застрявшему на середине агенту.
